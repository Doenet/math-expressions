# math-expressions-rs-wasm

The WebAssembly boundary for the [`math-expressions`](../math-expressions-rs)
Rust core, for JavaScript consumers ‚Äî principally **Doenet**, which plots via
**jsxgraph**. This package co-locates two things (the Doenet
`doenetml-worker-rust` layout):

1. **`crate/`** ‚Äî the `math-expressions-wasm` Rust crate: a thin `wasm-bindgen`
   adapter over the core crate's public API, compiled to a `cdylib`. Build it
   with `crate/build-wasm.sh` (or `npm run build:wasm`), which emits the
   wasm-bindgen package into `pkg/` (git-ignored) and runs a node smoke test.
2. **`src/*.ts`** ‚Äî the TypeScript bindings (the AST ‚Üí math.js bridge below).

## What's here

The load-bearing TS piece is the **AST ‚Üí math.js bridge**. The Rust port owns the
AST and its normalization; this package owns only the construction of math.js
nodes (which are JavaScript objects that must be built in JS). A consumer
compiles an expression **once** and then calls `.evaluate(scope)` per sample in
a tight loop that stays entirely in JS ‚Äî no JS‚ÜîWASM boundary crossing per point,
which is what makes numeric graphing fast.

This is "option 1" of the porting analysis: keep AST ‚Üí math.js in JS/TS, fed the
already-normalized Rust AST, rather than reimplementing math.js in Rust.

## Layout

- `crate/` ‚Äî the `math-expressions-wasm` Rust crate (`src/*.rs`,
  `Cargo.toml`, `build-wasm.sh`, `wasm-smoke.cjs`). A workspace member built for
  `wasm32-unknown-unknown`; see its `src/lib.rs` for the exported surface.
- `pkg/` ‚Äî wasm-bindgen output (generated by `build-wasm.sh`, git-ignored).
- `src/tree-to-mathjs.ts` ‚Äî `Tree` (the JSON AST from `expr.tree_json()`) ‚Üí
  math.js `MathNode`, plus `factorial ‚Üí gamma`, `compileTree`, and the
  `rustExprToMathNode` / `compileRustExpr` bridge (normalization done Rust-side
  via `normalize_function_names`).
- `src/wasm.ts` ‚Äî structural types for the wasm module surface. The
  authoritative types are the generated `pkg/math_expressions_wasm.d.ts`; these
  are a minimal, build-independent subset.
- `src/index.ts` ‚Äî public entry point.

## Usage

```ts
import { create, all } from "mathjs";
import { compileRustExpr } from "math-expressions-rs-wasm";
// `expr` is a parsed handle from the wasm module (parse_text / parse_latex)

const math = create(all, {});
const compiled = compileRustExpr(math, expr); // normalizes Rust-side, builds node
const y = compiled.evaluate({ x: 3.2 });      // call this per sample (jsxgraph)
```

math.js is a **peer dependency** ‚Äî pass in your own configured instance so it
isn't double-bundled.
